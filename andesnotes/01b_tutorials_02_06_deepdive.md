## 02 First Simulation

### Shortest closed loop (5 minutes)
1. `andes.load(get_case(...))` Load case (System object)
2. `ss.PFlow.run()` Establish steady-state operating conditions
3. `ss.TDS.config.tf = ...; ss.TDS.run()` for transient simulation
4. `ss.TDS.plt.plot(ss.GENROU.omega)` See the speed response
5. Create a new system `ss2` and then do `EIG.run()` (to avoid TDS rewriting status from affecting eigenvalue analysis)

### Key understanding
- `System` is a unified container: models, parameters, routines, and outputs are all in it.
- The output files are generated by default: `*_out.txt`, `*_out.lst`, `*_out.npz`.
- The CLI corresponds one-to-one with the Python interface: `andes run/plot`.

## 03 Power Flow Analysis

### Solver and Convergence
- Method: Newton-Raphson
- Common iterations: 3~6 times
- Key configuration:
  - `max_iter` (default 25)
  - `tol` (default 1e-6)
  - `method='NR'`
  - `init_tds` (whether to initialize the dynamic model in conjunction with each other)

### Result reading
- Bus voltage amplitude: `ss.Bus.v.v`
- Bus angle: `ss.Bus.a.v`
- Generator active/reactive power: `ss.PV.p.v`, `ss.PV.q.v`, `ss.Slack.p.v`
- Line flow: `ss.Line.a1.v`, `ss.Line.a2.v`
- DataFrame full table: `ss.Bus.cache.df`

### Troubleshooting Framework
- Non-convergence usually results from:
  1) Topology/data issues (islands, zero impedance, etc.)
  2) Infeasible working conditions (power balance/insufficient reactive power support)
  3) Initial value is too bad
- Methods: Increase `max_iter`, enable PV-PQ conversion, and open DEBUG to see the residual trajectory.

## 04 Time-Domain Simulation

### Core Mechanism
- Solution object: DAE
- Numerical method: implicit trapezoidal (A-stable)
- There must first be a convergence trend

### Disturbance modeling three-piece set
1. `Fault`: Three-phase fault (`bus`, `tf`, `tc`, `rf`, `xf`)
2. `Toggle`: Equipment removal/reclosure (line, unit, load, etc.)
3. `Alter`: Change parameters/settings during simulation (`src`, `method`, `amount`)

### Very critical engineering details
- It is usually necessary to load `setup=False` before adding perturbation, and `setup()` after modification.
- The dynamic model will take over the corresponding static model (such as `GENROU` taking over `PV/Slack`).
  - Therefore, changing `PV.p0` during simulation often has no effect; you should change `TGOV1.pref0/paux0`.

### Common configuration
- `tf`, `tstep`, `max_iter`, `tol`, `fixt`
- Timing results: `ss.dae.ts.t/x/y`
- Address mapping: variable `a` attribute (state takes `x`, algebra takes `y`)

### Common troubleshooting
- TDS step does not converge: reduce `tstep`, increase `max_iter`
- First `TDS.init()` does initialization check
- `--flat` does a flat run check

## 05 Data and File Formats

### Supported formats
- ANDES native `.xlsx` (recommended)
- PSS/E `.raw + .dyr`
- Other interoperability formats (separate chapter of the document)

### Data reading and rewriting
- Read: `andes.load(...)`
- View the parameter table: `model.cache.df` (after pu) / `model.cache.df_in` (original input)
- Read single parameters: `model.get('param', idx)`
- Change parameters: `model.alter('param', idx, value)`
- Batch modification: directly modify `param.v[:]`
- After changing, you need to use `cache.refresh()` to update the view.

### Index system
- External index: `idx` (string optional)
- Internal location: `uid` (0-based)
- Conversion: `idx2uid()`

### Save
- `andes.io.xlsx.write(system, 'xxx.xlsx', overwrite=True)`

## 06 Plotting Results

### Plotter object
- `ss.TDS.plt` (if empty, `ss.TDS.load_plotter()`)

### Two drawing entrances
1. By variable object: `plot(ss.GENROU.omega)` (most recommended)
2. By index: `plot((5,6,7,8))` (consistent with CLI)
   - Index search: `ss.TDS.plt.find('omega')`

### Common parameters
- `a` (subset selection)
- `ylabel`, `yheader`
- `ycalc` (unit conversion, such as pu→Hz)
- `grid`, `greyscale`
- `savefig`, `dpi`
- `fig`, `ax` (superimposed curve)

### Data export
- Full CSV: `ss.TDS.plt.export_csv('x.csv')`
- Custom export: directly spell `ss.dae.ts` + pandas

## Phased summary (02~06)
- The ANDES tutorial is not a "function demonstration", but a complete project chain:
  **Data -> Power flow -> Transient -> Visualization -> Export -> Troubleshooting**.
- The most valuable tacit knowledge for practice:
  1) Dynamic/static model takeover relationship
  2) setup timing and event injection sequence
  3) Address system (variable object vs index)
